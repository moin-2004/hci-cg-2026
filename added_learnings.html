Self Study

Effective interface design requires time and careful consideration, but even small, user-focused decisions—such as involving real users early in the process—can help avoid major design problems. The automatic syringe case illustrates how a numeric keypad interface caused a dangerous tenfold dosage error, an issue that was only identified after consulting hospital staff.

Analyzing the Syringe Interface Using Different Models

The sections below describe how Norman’s Model, GOMS, and KLM each evaluate the syringe interface redesign, in which a numeric keypad was replaced with increment and decrement buttons.

Norman’s Cycle of Action

Norman’s model reveals a significant gulf of execution in the original keypad-based design. The user’s goal of setting a precise dose was poorly supported, as a single extra key press could lead to a lethal dosage. The redesigned button interface narrows this gulf by aligning user intentions with safer, more controlled actions.

GOMS Model

From a GOMS perspective, entering a dose is decomposed into goals, methods, and operators. While the keypad allows quicker input, it offers little assistance for spotting or correcting errors. The button-based approach is slower, but it requires users to confirm each change, increasing overall task safety.

Keystroke-Level Model (KLM)

KLM is used to predict execution time for experienced users. It would likely indicate that keypad entry is faster due to fewer keystrokes, whereas repeated button presses take more time. However, this comparison also highlights that efficiency alone is insufficient when evaluating safety-critical systems such as medical devices.

The Importance of Iteration

Because users and usage contexts are complex, interface design rarely succeeds on the first attempt. Developing early prototypes and engaging users throughout the design process helps uncover serious issues before they become permanently built into the system.

Norman’s Model of Interaction

Donald Norman is a leading figure in Human–Computer Interaction, widely recognized for advocating user-centered design, usability, and cognitive engineering. His interaction model is influential because it closely reflects how users naturally think about and interact with computer systems.

In Norman’s model, interaction begins with the formation of a goal and a plan to achieve it. The user then carries out actions through the interface. Afterward, the user observes the system’s response and evaluates whether the result matches the original goal, determining the next steps.

The Seven Stages of Action

Norman divides interaction into two primary phases: execution and evaluation. Together, these phases consist of seven stages:

  1. Defining the goal
  2. Forming the intention
  3. Specifying the action sequence
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome against the goal

Gulfs of Execution and Evaluation

Norman uses this framework to explain why poorly designed interfaces cause user difficulties. The gulf of execution is the gap between a user’s goals and the actions the system allows. The gulf of evaluation is the gap between the system’s output and how easily the user can understand it.

Human Error: Slips and Mistakes

Human errors can be categorized as slips or mistakes. Slips occur when the user understands the system but performs an incorrect action, such as pressing the wrong button. Mistakes occur when the user forms an incorrect goal due to misunderstanding the system, such as misreading the meaning of an icon.

Slips can often be minimized through improved interface design, such as increasing the distance between buttons. Mistakes usually require more substantial solutions, including interface redesigns, better training, or clearer metaphors.

GOMS Model

The GOMS model, introduced by Card, Moran, and Newell, explains task performance by breaking interaction into Goals, Operators, Methods, and Selection rules. It helps designers analyze both the mental and physical components of user tasks.

Goals define what the user aims to accomplish. Operators are basic actions, such as pressing a key or reading a message. Methods represent different ways to achieve a goal, while selection rules determine which method is chosen based on context or experience.

Design Focus: Why GOMS Saves Money

GOMS has been applied in real-world system evaluations. For instance, the telecommunications company NYNEX used GOMS to assess a proposed computer system and found it would perform more slowly than the existing one. As a result, the project was abandoned, saving the company millions of dollars.

Keystroke-Level Model (KLM)

The Keystroke-Level Model is a more detailed extension of GOMS that focuses on predicting execution time for expert users performing routine tasks. It is intended for short, clearly defined tasks that typically take less than 20 seconds.

KLM assumes error-free performance and focuses exclusively on the execution phase of interaction. It breaks tasks down into low-level operators, such as keystrokes, mouse clicks, pointing, hand movements, mental preparation, and system response time.

By assigning standard time values to each operator and adding them together, KLM enables designers to compare interface designs or task methods and predict which will be faster—without conducting full user studies.

Lecture 1

HCI Overview

Human–Computer Interaction (HCI) combines concepts from Cognitive Psychology, Design, and Computer Science to understand how humans interact with technology.

Human Behavior and Memory

  • Humans often perform tasks based on how they were taught or what they observed at a young age (e.g., drawing a glass from memory).
  • Habits can be changed, but the fundamental nature of a person remains stable.
  • Humans store information in:
    • Short-Term Memory (STM)
    • Long-Term Memory (LTM)
    • Perceptual (Sensory) Storage

Interfaces and Models

  • Types of interfaces: GUI, CLI, voice-based, gesture-based
  • Cognitive Neural Networks (CNNs) resemble human processing by recognizing patterns such as texture, shape, and color.
  • Fitts’s Law predicts the time required to move to a target based on distance and size and influenced the invention of the mouse.
  • Chunking breaks large information into smaller units (e.g., 1234567890 → 123-456-7890).

UI vs UX

UI (User Interface) focuses on the visual design of a product and how it looks.

UX (User Experience) focuses on the overall experience and how the product feels to use, including usability and satisfaction.

The Automatic Syringe case demonstrates how small interface design decisions can lead to serious consequences. Numeric keypads caused dangerous input errors due to accidental key presses.

Interfaces rarely work perfectly on the first attempt, making iteration essential. Early prototypes should be easy to modify or discard, and rapid prototyping techniques such as paper sketches and mock-ups help designers test ideas quickly before full development.
